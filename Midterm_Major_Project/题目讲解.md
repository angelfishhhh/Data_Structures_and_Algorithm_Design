# 题目一：分治算法 - 第 k 小元素选择

## 1. 问题说明

给定无序线性序列中的 `n` 个元素和整数 `k`，其中 `1 <= k <= n`，要求找出这 `n` 个元素中的第 `k` 小元素。

程序文件：

```text
src/selection.cpp
```

编译后可执行文件：

```text
bin/selection
```

## 2. 输入格式

第一行或前两个数字为：

```text
n k
```

后面输入 `n` 个整数：

```text
a1 a2 a3 ... an
```

示例：

```text
8 3
5 1 3 3 2 8 7 4
```

## 3. 运行方法

```bash
./bin/selection < tests/selection_sample.txt
```

也可以手动输入：

```bash
./bin/selection
```

然后输入：

```text
8 3
5 1 3 3 2 8 7 4
```

## 4. 整体设计思想

本题实现 3 种求第 `k` 小元素的方法，并把 3 种结果放在一起比较：

1. **基于最大堆的选择**：维护一个容量为 `k` 的最大堆，堆中始终保存当前扫描过的最小 `k` 个元素。扫描结束后，堆顶就是第 `k` 小元素。
2. **随机划分线性选择**：类似快速选择，每次随机取一个枢轴，把数组分成小于、等于、大于枢轴的 3 段，只递归或迭代进入包含第 `k` 小元素的那一段。
3. **中位数的中位数线性选择**：每 5 个元素分组，取每组中位数，再递归选择这些中位数的中位数作为枢轴，使最坏情况下也能保持线性时间复杂度。

程序最后比较 3 种算法的结果是否一致。如果一致，说明实现结果相互验证通过。

## 5. 函数说明

### `int heapSelect(const vector<int>& values, int k)`

作用：使用最大堆求第 `k` 小元素。

具体过程：

1. 取前 `k` 个元素建立最大堆。
2. 从第 `k+1` 个元素开始扫描。
3. 如果当前元素小于堆顶，说明它应该进入最小的 `k` 个元素集合，于是弹出堆顶并插入当前元素。
4. 扫描结束后，堆顶就是第 `k` 小元素。

复杂度：

```text
时间复杂度：O(k + (n-k)logk)
空间复杂度：O(k)
```

### `pair<int, int> partitionThreeWay(vector<int>& values, int left, int right, int pivotValue)`

作用：对 `values[left..right]` 做三路划分。

划分后数组被分成 3 段：

```text
[left, lt-1]      小于 pivotValue
[lt, gt]          等于 pivotValue
[gt+1, right]     大于 pivotValue
```

返回值 `{lt, gt}` 表示等于枢轴值的区间边界。

为什么要三路划分：输入中可能有重复元素。如果只做普通二路划分，遇到大量重复值时，递归区间可能缩小很慢；三路划分可以一次排除所有等于枢轴的元素。

### `int randomizedSelect(vector<int> values, int k)`

作用：使用随机划分选择算法求第 `k` 小元素。

注意：参数 `values` 是值传递，因此函数内部修改数组不会影响原数组。

具体过程：

1. 把第 `k` 小转换为 0 下标位置 `target = k - 1`。
2. 在当前区间 `[left, right]` 中随机取枢轴。
3. 调用 `partitionThreeWay` 得到等于枢轴的区间 `[lt, gt]`。
4. 如果 `target < lt`，说明答案在左边。
5. 如果 `target > gt`，说明答案在右边。
6. 如果 `lt <= target <= gt`，说明枢轴值就是第 `k` 小元素。

复杂度：

```text
平均时间复杂度：O(n)
最坏时间复杂度：O(n^2)
空间复杂度：O(n)，因为传入时复制了一份数组
```

程序中随机数种子固定为 `20260518`，这样每次实验结果更容易复现。

### `int medianOfMedians(vector<int>& values, int left, int right, int target)`

作用：中位数的中位数算法核心递归函数。

参数含义：

```text
values  待选择数组
left    当前处理区间左端点
right   当前处理区间右端点
target  要找的 0 下标位置
```

具体过程：

1. 如果当前区间元素个数不超过 5，直接排序并返回目标位置元素。
2. 否则每 5 个元素分成一组。
3. 每组内部排序，取该组中位数。
4. 把各组中位数移动到数组前部。
5. 递归求这些中位数的中位数，作为本轮枢轴。
6. 对当前区间做三路划分。
7. 根据 `target` 所在位置，只进入左段或右段继续查找；如果落在等于枢轴的区间，直接返回枢轴值。

复杂度：

```text
最坏时间复杂度：O(n)
空间复杂度：递归栈空间
```

### `int medianOfMediansSelect(vector<int> values, int k)`

作用：对外包装函数，把用户传入的 `k` 转换成 0 下标，并调用 `medianOfMedians`。

同样使用值传递复制数组，避免修改原始输入。

### `template <typename Function> pair<int, long long> timedSelect(Function function)`

作用：统计某个选择算法的运行时间。

返回：

```text
first   算法得到的第 k 小元素
second  运行时间，单位为微秒 us
```

### `int main()`

作用：程序入口。

完成以下工作：

1. 读取并检查 `n`、`k`。
2. 读取 `n` 个整数。
3. 分别调用 3 种选择算法。
4. 输出 3 种算法的答案和运行时间。
5. 判断 3 种算法结果是否一致。

如果输入非法，例如 `k < 1` 或 `k > n`，程序会输出错误信息并返回非 0 状态码。

## 6. 输出结果含义

样例输出：

```text
Input size: 8
k: 3
HeapSelect: 3 (time_us: 0)
RandomizedSelect: 3 (time_us: 1)
MedianOfMediansSelect: 3 (time_us: 0)
All algorithms agree: yes
```

含义：

```text
Input size: 8                         输入规模 n 为 8
k: 3                                  要找第 3 小元素
HeapSelect: 3                         堆选择算法结果为 3
RandomizedSelect: 3                   随机选择算法结果为 3
MedianOfMediansSelect: 3              中位数的中位数算法结果为 3
time_us                               对应算法耗时，单位为微秒
All algorithms agree: yes             3 种算法结果一致
```

# 题目二：动态规划 - 卡牌价值二分

## 1. 问题说明

给定 `n` 张卡牌，每张卡牌有正整数价值。要求把卡牌分成两堆，使两堆价值和的差值尽可能小，并输出两堆的价值和与卡牌编号。

程序文件：

```text
src/partition.cpp
```

编译后可执行文件：

```text
bin/partition
```

## 2. 输入格式

输入一行正整数，表示每张卡牌的价值：

```text
v1 v2 v3 ... vn
```

示例：

```text
2 1 3 1 5 2 3 4
```

卡牌编号按照输入顺序从 `1` 开始编号。

## 3. 运行方法

```bash
./bin/partition < tests/partition_sample.txt
```

也可以手动输入：

```bash
./bin/partition
```

然后输入：

```text
2 1 3 1 5 2 3 4
```

## 4. 整体设计思想

本题实现了两种方法：

1. **暴力枚举法**：枚举所有子集，把一个子集作为 A 堆，其余卡牌作为 B 堆，计算两堆差值，保留差值最小的方案。
2. **动态规划法**：把问题转化为“从所有卡牌中选出一部分，使其价值和尽量接近总价值的一半”。设总价值为 `total`，只需要找到不超过 `total / 2` 的最大可达价值 `s`，则两堆差值为 `total - 2s`。

暴力法用于小规模数据验证正确性；动态规划法用于更高效地求解。

## 5. 数据结构说明

### `struct Division`

作用：保存一次分堆方案。

字段含义：

```text
sumA        A 堆价值总和
sumB        B 堆价值总和
difference 两堆价值差的绝对值
pileA       A 堆中的卡牌编号
pileB       B 堆中的卡牌编号
```

## 6. 函数说明

### `Division buildDivision(const vector<int>& values, const vector<int>& pileA)`

作用：根据 A 堆的卡牌编号生成完整的分堆结果。

具体过程：

1. 根据 `pileA` 标记哪些卡牌在 A 堆。
2. 计算 A 堆价值和 `sumA`。
3. 用总价值减去 `sumA` 得到 B 堆价值和 `sumB`。
4. 计算差值 `abs(sumA - sumB)`。
5. 把没有进入 A 堆的卡牌编号放入 B 堆。

返回一个完整的 `Division` 结构。

### `Division bruteForcePartition(const vector<int>& values)`

作用：用暴力枚举法求最优分堆。

具体过程：

1. 对 `n` 张卡牌，一共有 `2^n` 个子集。
2. 用二进制掩码 `mask` 表示一个子集。
3. 如果 `mask` 的第 `i` 位为 1，表示第 `i+1` 张卡牌进入 A 堆。
4. 枚举每个 `mask`，计算 A 堆价值和。
5. 根据 `abs(total - 2 * sumA)` 计算两堆差值。
6. 保存差值最小的方案。

复杂度：

```text
时间复杂度：O(n * 2^n)
空间复杂度：O(n)
```

代码中当差值相同时，会优先保留 A 堆价值较小的一种方案。这不影响最优差值，只是让输出更稳定。

### `Division dynamicProgrammingPartition(const vector<int>& values)`

作用：用动态规划求最优分堆。

状态定义：

```text
dp[i][s] = 使用前 i 张卡牌时，是否能组成价值和 s
```

状态转移：

```text
dp[i][s] = dp[i-1][s] || dp[i-1][s - values[i-1]]
```

其中：

```text
dp[i-1][s]                     不选第 i 张卡牌
dp[i-1][s - values[i-1]]        选择第 i 张卡牌
```

具体过程：

1. 计算总价值 `total`。
2. 目标上界为 `target = total / 2`。
3. 建立二维布尔表 `dp[n+1][target+1]`。
4. 初始化 `dp[0][0] = true`。
5. 逐张卡牌更新可达价值。
6. 从 `target` 向下寻找最大可达价值 `best`。
7. 从 `dp` 表中回溯出组成 `best` 的卡牌编号，作为 A 堆。
8. 调用 `buildDivision` 得到完整分堆结果。

复杂度：

```text
时间复杂度：O(nS)
空间复杂度：O(nS)
```

其中 `S = total / 2`。

### `template <typename Function> pair<Division, long long> timedPartition(Function function)`

作用：执行某个分堆算法并统计耗时。

返回：

```text
first   分堆结果 Division
second  运行时间，单位为微秒 us
```

### `void printIndices(const vector<int>& indices)`

作用：输出某一堆中的卡牌编号。

如果这一堆为空，输出：

```text

```

否则按空格分隔输出编号。

### `void printDivision(const string& name, const Division& division, long long timeUs)`

作用：按照统一格式输出某个算法的分堆结果。

输出内容包括：

```text
算法名称 best difference
算法名称 pile A sum
算法名称 pile A indices
算法名称 pile B sum
算法名称 pile B indices
算法名称 time_us
```

### `int main()`

作用：程序入口。

完成以下工作：

1. 读取一行或多行正整数卡牌价值，直到输入结束。
2. 检查所有价值是否为正整数。
3. 输出卡牌数量和总价值。
4. 当 `n <= 22` 时运行暴力法；如果 `n > 22`，跳过暴力法，避免 `2^n` 枚举过慢。
5. 运行动态规划法。
6. 输出两种方法的结果和耗时。

## 7. 输出结果含义

# 题目三：贪心算法 - 供水管网布局

## 1. 问题说明

有 `n` 栋居民楼，楼栋之间可铺设供水管道，每条管道有不同成本。要求：

1. 所有楼栋连通。
2. 管网不能出现环路。
3. 总管道成本最小。

这正是无向带权图的最小生成树问题。程序使用 Kruskal 算法实现。

程序文件：

```text
src/mst.cpp
```

编译后可执行文件：

```text
bin/mst
```

## 2. 输入格式

第一行：

```text
n m
```

含义：

```text
n  楼栋数量，编号为 1..n
m  可铺设管道数量
```

接下来 `m` 行，每行一条边：

```text
u v w
```

含义：

```text
u  管道一端楼栋编号
v  管道另一端楼栋编号
w  管道成本或长度
```

示例：

```text
6 9
1 2 6
1 3 1
1 4 5
2 3 5
2 5 3
3 4 5
3 5 6
3 6 4
4 6 2
```

## 3. 运行方法

```bash
./bin/mst < tests/mst_sample.txt
```

也可以手动输入：

```bash
./bin/mst
```

然后输入图数据。

## 4. 整体设计思想

Kruskal 算法是一种贪心算法：

1. 把所有管道按成本从小到大排序。
2. 从成本最小的管道开始尝试选择。
3. 如果选择这条管道不会形成环路，就加入方案。
4. 如果会形成环路，就跳过。
5. 当选中 `n - 1` 条管道时，所有楼栋已经连通，且没有环路，得到最小生成树。

为了快速判断是否成环，程序使用并查集维护连通分量。

## 5. 数据结构说明

### `struct Edge`

作用：表示一条候选供水管道。

字段含义：

```text
u  管道一端楼栋编号
v  管道另一端楼栋编号
w  管道成本或长度
```

### `class DisjointSet`

作用：并查集，用于 Kruskal 算法中判断两个楼栋当前是否已经连通。

成员变量：

```text
parent  每个节点的父节点
rank    按秩合并时使用的树高度近似值
```

## 6. 函数说明

### `explicit DisjointSet(int n)`

作用：初始化并查集。

初始化时每个楼栋单独属于一个集合：

```text
parent[i] = i
rank[i] = 0
```

### `int find(int x)`

作用：查找节点 `x` 所在集合的代表节点。

实现中使用路径压缩：

```text
如果 parent[x] != x，就递归查找根节点，并直接把 parent[x] 改为根节点。
```

路径压缩可以加快后续查找速度。

### `bool unite(int a, int b)`

作用：尝试合并节点 `a` 和 `b` 所在集合。

返回值含义：

```text
true   原来不在同一个集合，合并成功，说明选择这条边不会成环
false  原来已经在同一个集合，不能合并，说明选择这条边会成环
```

实现中使用按秩合并，使并查集树尽量保持较低高度。

### `int main()`

作用：程序入口。

完成以下工作：

1. 读取 `n`、`m`。
2. 读取所有边 `u v w`。
3. 检查楼栋编号和边权是否合法。
4. 按边权从小到大排序；边权相同时按 `u`、`v` 排序，使输出更稳定。
5. 依次扫描边，调用 `dsu.unite(u, v)` 判断是否加入最小生成树。
6. 统计选中边和总成本。
7. 如果最终选中边数不是 `n - 1`，说明图不连通，无法形成覆盖所有楼栋的供水管网。
8. 输出最小生成树结果。

## 7. 输出结果含义

样例输出：

```text
Buildings: 6
Total cost: 15

```


含义是这些候选管道无法把所有楼栋连接成一个整体供水网络。
